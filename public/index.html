SHARP Context — How it works

When Prompt Opinion calls any tool on this server, it automatically fires three HTTP headers with the patient's FHIR context. The server reads them and queries the FHIR server on behalf of the agent — no manual auth code required.

X-FHIR-Server-URL
Base URL of the FHIR server (Epic, Cerner, HAPI, etc.)
X-FHIR-Access-Token
Bearer token with patient-scoped SMART on FHIR permissions
X-Patient-ID
The FHIR patient resource ID used to scope all queries

Example — list available tools

curl -X POST https://fhir-mcp-server.onrender.com/mcp \
  -H "Content-Type: application/json" \
  -H "X-FHIR-Server-URL: https://hapi.fhir.org/baseR4" \
  -H "X-FHIR-Access-Token: open" \
  -H "X-Patient-ID: example-patient-123" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'
